# 006 Rules Engine - Requirements

## Goal

Provide deterministic, testable D&D 5e-compatible core rule resolution for the MVP.

The rules engine owns dice, checks, saves, attacks, damage, healing, conditions, initiative, and combat turn advancement.

## User stories

### Story 1: Dice are system-generated

As a player, I want dice rolls to be generated by the system, so that the AI DM cannot invent results.

#### Acceptance criteria

- WHEN a roll is requested THEN the rules engine rolls dice.
- WHEN a roll completes THEN the result includes formula, raw dice, modifiers, total, and success/failure where applicable.
- WHEN replaying events THEN dice are not rerolled.

### Story 2: Ability and skill checks are calculated

As a player, I want ability and skill checks to use my character stats, so that results match 5e expectations.

#### Acceptance criteria

- WHEN a character rolls an ability check THEN the ability modifier is applied.
- WHEN a proficient skill is used THEN proficiency bonus is applied.
- WHEN advantage/disadvantage is set THEN the correct d20 is selected.
- WHEN total meets or exceeds DC THEN the check succeeds.

### Story 3: Combat basics work

As a Host, I want attacks, damage, HP, and initiative to be resolved by code, so that combat state remains consistent.

#### Acceptance criteria

- WHEN combat starts THEN combatants receive initiative order.
- WHEN an attack roll equals or exceeds target AC THEN it hits.
- WHEN damage is applied THEN HP changes deterministically.
- WHEN HP reaches 0 THEN the target state reflects being defeated/unconscious as appropriate.

## Functional requirements

1. Parse and roll common dice formulas such as `1d20`, `2d6+3`, `1d8 + 4`.
2. Support injected RNG for tests.
3. Calculate 5e ability modifiers.
4. Resolve ability checks, skill checks, saving throws, attacks, damage, healing, and conditions.
5. Produce results that can be serialized into SessionEvents.
6. Avoid any dependency on LLM providers.

## Non-goals

- Full 5e automation.
- Complete spell automation.
- Complete class feature automation.
- Grid movement.
- Complex reactions and interrupts.
- Exhaustive monster AI.
