# MVP-1.0 Playtest Checklist

## Setup

- Confirm branch, version, and commit under test.
- Confirm no secrets are committed.
- Confirm default automated tests make no live provider calls.
- Confirm AI provider config is disabled by default.
- When running a live AI provider, record AI provider endpoint status, model, and config source without recording secrets.

## Participants

- Host:
- Player 1:
- Player 2:
- Optional Player 3:
- Optional Player 4:

## Adventure

- Use `The Lantern Beneath the Hill`.
- Confirm the adventure source is the original TableMind fixture.
- Confirm no non-SRD commercial adventure content is loaded.

## Host Controls

- Create room and copy invite link.
- Load demo adventure.
- Start session.
- Inspect DM-only scene notes.
- Reveal at least one clue.
- Change scene at least once.
- Pause and resume AI.
- Approve, edit, or reject at least one AI review item if review is triggered.
- Complete the session.

## AI Provider And Spoiler Watch

- Confirm AI provider enabled or disabled status.
- Confirm low-confidence output enters Host review.
- Confirm reveal proposals enter Host review.
- Confirm state patch proposals enter Host review.
- Confirm spoiler incidents are recorded with the blocked phrase/entity.
- Confirm safe AI narration can become a public AI message only after validation.

## Rules And Combat

- Resolve at least one skill check, ability check, or saving throw through deterministic rules code.
- Confirm the AI does not fabricate dice.
- Start one combat encounter.
- Resolve at least one player attack through deterministic rules code.
- End combat through Host control.

## Recap

- Generate player recap.
- Generate Host recap.
- Confirm player recap excludes DM-only truth and rejected AI output.
- Confirm Host recap includes unresolved Host-only threads.

## Exit

- Record participants, adventure, duration, AI provider status, spoiler incidents, Host interventions, rules outcomes, combat outcomes, blockers, and recap status in the report.
- Record the Pass/fail decision.
